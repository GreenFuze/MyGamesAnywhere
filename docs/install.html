Windows release

Install MGA as portable, for-me-only, or all-users mode.

MGA v0.0.8 ships as a Windows portable ZIP and a Windows installer. It runs locally, keeps portable installs self-contained, moves installed mutable data and logs out of the app directory, and keeps portable/for-me installs on 127.0.0.1:8900.

  1. Download mga-v0.0.8-windows-amd64-portable.zip or mga-v0.0.8-windows-amd64-installer.exe from GitHub Releases.
  2. For portable, extract it into a writable folder such as C:\Games\MGA and run Start MGA.cmd.
  3. For installer, choose for-me-only mode for a login process or all-users mode for a Windows service.
  4. Open http://127.0.0.1:8900.
  5. Create or select a local profile in the first-run/profile picker flow.

Local-first runtime

The current package behaves more like a self-hosted game library running on your own machine than a cloud-first account shell.

  • local server
  • local database
  • local media cache
  • local config
  • plugin-backed integrations

Portable build notes

Avoid extracting the portable build under Program Files. Portable runtime writes config, database, media, logs, update cache, and local state inside the portable folder. The shipped config uses LISTEN_IP: "127.0.0.1" and PORT: "8900"; LAN binding is opt-in by editing config.json.

Installer notes

For-me-only installs place app files under %LOCALAPPDATA%\Programs\MyGamesAnywhere and data/logs under %LOCALAPPDATA%\MyGamesAnywhere; MGA starts after that Windows user signs in and listens only on this PC by default. All-users installs place app files under %ProgramFiles%\MyGamesAnywhere and data/logs under %ProgramData%\MyGamesAnywhere; MGA runs as a Windows service, requires administrator approval, starts even when nobody is signed in, and enables LAN access by default.

Portable upgrade policy

  1. Use Settings -> Update to check the release manifest.
  2. MGA downloads and verifies the portable ZIP with SHA256.
  3. The portable updater restarts MGA, replaces app files, and preserves config.json, data/, media/, source_cache/, updates/, and logs/.
  4. Verify the version in Settings after restart.

Auto-update v1

Settings -> Update can check a release manifest, download the matching asset, and verify SHA256. Installed Windows layouts launch the verified installer with silent update arguments. Portable Windows layouts launch an external updater script so the running server never overwrites itself.
