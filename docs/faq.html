FAQ

Questions people search for before trying MGA.

Can MGA merge Steam, Xbox, Epic, and emulator games into one library?

Yes. MGA is built around canonical game records that can merge source entries when the detection flow can prove they represent the same game.

Does MGA support ROMs and emulators?

Yes. ROM and emulator-style collections are part of the product direction alongside storefront and cloud-source workflows.

Does MGA work offline?

The local runtime works offline. Some metadata providers, source integrations, achievements, and sync targets require network access.

Where does MGA store data?

The current portable runtime stores config, database, media, plugins, and frontend files beside the runtime folder.

Can MGA detect games on SMB shares or external drives?

Yes. SMB/network shares and removable/local paths are core source scenarios.

Does MGA support browser play?

Yes where configured runtime support exists. MGA can launch supported browser runtimes such as EmulatorJS, js-dos, and ScummVM from the web UI.

Can MGA sync saves?

Yes. Local disk and Google Drive save-sync integrations are available in the current plugin set.

Is MGA Windows-only right now?

The first packaged release is Windows-only and portable. Cross-platform installers are planned later.
