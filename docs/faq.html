FAQ

Questions people ask before trying MGA.

Is MGA a Playnite alternative?

It can be, depending on what you want. If you care most about canonical merge, provenance, manual repair, and a local web/API surface, MGA is aimed at that problem. If you mainly want a polished storefront shell, Playnite may still feel closer today.

Is MGA a LaunchBox alternative?

Partly. MGA overlaps with LaunchBox-style mixed-library organization, but its strongest differentiator is not themeing or cabinet presentation. It is source-backed canonical game identity, local ownership, and visible repair workflow.

Does MGA replace Steam, Xbox, or Epic?

No. MGA sits above those systems as a local game library and launcher layer. It consumes source and provider data; it does not replace the services themselves.

Does MGA store my data locally?

Yes. MGA is local-first. The database, media cache, config, and plugins live on your machine unless you explicitly configure sync integrations.

Can MGA handle ROMs and emulators?

Yes. ROM and emulator-style collections are part of the core product direction alongside storefront, cloud, and file-backed sources. Browser-play surfaces include EmulatorJS, js-dos, and ScummVM where configured.

Can one canonical game have multiple playable versions?

Yes. MGA keeps concrete source records under the canonical game and exposes source-backed launch options so clients can choose the specific source/version to run.

Can achievements differ by source or platform?

Yes. Achievement sets are source-backed, so a canonical game can show multiple sets such as an arcade version and a console version when both are detected.

How does browser play work with cloud or network files?

Remote and network-backed files can be materialized into a local server cache before launch, so browser runtimes do not have to stream many small files directly from Drive or SMB during play.

Does save sync work inside EmulatorJS?

MGA can hook EmulatorJS save-state and save-RAM actions through the server-side save-sync cache when a save-sync integration is active.

What happens when detection is wrong?

MGA keeps unresolved or questionable records visible in a manual review workflow. You can re-detect, search configured metadata providers with fuzzy platform-aware matching, apply a better match, or archive a false positive.

Can MGA delete false-positive source files?

For file-backed sources, yes. MGA asks the source plugin for a dry-delete preview, displays the exact returned targets, and requires a checkbox before enabling the real delete action. Google Drive files move to Drive trash; SMB deletes only explicit file targets.

Can I inspect where a game came from?

Yes. Canonical pages keep source records, files, root paths, resolver matches, provider links, and other provenance visible.

Does MGA support Game Pass and xCloud launch surfaces?

MGA can surface Game Pass and xCloud availability through Xbox-backed data where the connected source exposes it.

Does MGA expose an API?

Yes. MGA exposes a local REST API and a React frontend over the same local library.

Is MGA Windows-only right now?

The current packaged release is Windows-first and includes a portable ZIP plus an installer. Cross-platform installers are planned later.

Can MGA listen on my LAN?

The server supports LISTEN_IP in config.json, including loopback, a concrete LAN IP, and 0.0.0.0. Portable releases ship local-only on 127.0.0.1:8900 by default.
