# v0.0.10 Release Notes

`v0.0.10` continues the Windows portable and installer line with a focus on library ownership, achievement refresh reliability, duplicate cleanup, and manual repair controls.

## Highlights

- Added Library Statistics and Gamer Statistics pages for profile-scoped library and achievement overview.
- Made Play the default app landing page and removed the duplicate Home dashboard.
- Reworked Achievements into provider-aware tabs, stored-achievement summaries, provider-specific points, cleaner refresh warnings, and background refresh progress.
- Added server-owned achievement refresh after scans, including persisted per-game/provider refresh state.
- Added conservative RetroAchievements background throttling and clean rate-limit handling instead of surfacing raw provider HTML.
- Hardened profile ownership for browser-play streaming, SSE events, and favorites.
- Added Settings -> Duplicates with duplicate review, cover/title links, batch hard-delete review, progress, and source-plugin delete previews.
- Added canonical split/merge controls so one source record can be separated from a wrongly merged canonical game or pinned into an existing canonical game.
- Improved direct reclassify so the selected metadata result replaces stale resolver matches and media.
- Improved packed Windows installer detection so filenames such as `setup_lego_batman_1.0_(18156)` use cleaned lookup variants and avoid wrong-platform automatic matches.
- Tightened automatic metadata consensus so known-platform source records, including SNES games, reject same-title wrong-platform media candidates.
- Improved media-cache diagnostics with recent errors and failing media URLs.
- Hid plugin subprocess console windows in packaged Windows builds.
- Made the game-detail floating Play action icon-only while keeping accessible labels.

## Upgrade Notes

- Database migrations are included up to migration `9`.
- Migration `7` adds `achievement_refresh_states` for persisted achievement refresh status.
- Migration `8` makes favorites profile-scoped while backfilling existing favorites to profiles that could see the game.
- Migration `9` adds `canonical_source_pins` for profile-scoped split/merge grouping decisions.
- Media assets and plugin binaries remain global; canonical media override rows are cleared by reclassify and split/merge operations when needed.
- Existing installs should update through the normal packaged updater flow, which backs up app files and the SQLite DB triplet before migration.
