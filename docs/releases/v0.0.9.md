# v0.0.9 Release Notes

`v0.0.9` is a follow-up Windows release for installed `v0.0.8` systems. It keeps the same profile, installer, and updater foundation while fixing the Steam source scan path, polishing first-run restore, and improving update progress/error feedback.

## Highlights

- Fixed Steam source scans in profile-owned installed setups. Steam status checks already used the profile-owned integration config, but scans still read the plugin process config and could fail with `steam source requires api_key before it can scan games`.
- Fixed Steam achievement fetches to use the same profile-owned config path as scans.
- Improved Settings -> Update:
  - download progress is streamed to the UI
  - downloaded assets are detected and verified on check
  - `Download` changes to `Re-download` after a verified asset exists
  - `Apply` shows a restart/waiting state and warns if the old server stays online
  - apply refuses missing or SHA-mismatched update files before launching the updater
- Improved the first-run restore flow:
  - hides raw provider sign-in URLs
  - makes the paste-callback guidance clearer for remote browsers
  - removes the hardcoded `Games/mga_sync` default from the restore path field
  - labels the encryption key as a restore step
  - sends restored users to Settings -> Integrations instead of the home page
  - hands the first restore scan to the existing Integrations scan-progress UI

## Upgrade Notes

- Installed `v0.0.8` systems should detect `v0.0.9` through Settings -> Update.
- No database migration is required for this release.
- Existing profile data, integrations, media, source cache, and settings are preserved by the updater.

## Validation Focus

- Update from installed service-mode `v0.0.8` to `v0.0.9`.
- Steam source status check and source scan after settings-sync restore.
- First-run restore via Google Drive settings sync on a remote browser.
- Settings -> Integrations scan-progress display immediately after first-run restore.
