# v0.1.2 Release Notes

`v0.1.2` is a metadata and media reliability patch release after `v0.1.1`.

## Highlights

- **Images now survive Refresh Metadata.** Provider images (covers, screenshots, backgrounds) were silently wiped on every Refresh Metadata click for manually-matched games. The root cause was that resolver-match media was never persisted to the database, so each refresh rebuilt the game from mediaisless records and cleared `source_game_media` with nothing to replace it. Media is now stored in `metadata_json` alongside the resolver match and restored on read, so refreshes correctly re-populate all images.
- **Manual reclassify preserves the full media list on subsequent refreshes.** The locked resolver match created when a user manually selects a game now carries the provider's complete media list (covers, screenshots, backgrounds, logos) instead of only the single representative image URL. This means the first Refresh Metadata after deploying v0.1.2 self-heals any game where images were previously lost.
- **Metadata refresh no longer re-downloads the LaunchBox index on every single-game refresh.** A `force: true` flag incorrectly caused the full ~100 MB LaunchBox `Metadata.zip` to be re-downloaded on every Refresh Metadata click. It now respects the 30-day local cache, so single-game refreshes are fast when the index is fresh.
- **Refresh Metadata can no longer be aborted by client disconnect.** The refresh work ran under the HTTP request context, meaning a browser timeout or navigating away mid-refresh could cancel the context and cause SQLite writes to fail with `context deadline exceeded`. Refresh now runs under a detached context with a 10-minute ceiling.
- **LaunchBox game detail links no longer point to the wrong game.** The LaunchBox `Metadata.xml` `DatabaseID` does not map to the `gamesdb.launchbox-app.com/games/details/{id}` URL scheme — the two numbering systems are independent. External links now use a title-based search URL so they always point to the correct game search results.

## Upgrade Notes

- `NO_MIGRATION_NEEDED`: v0.1.2 changes metadata persistence logic and refresh behavior only. No SQLite schema, persisted config, or sync payload changes are introduced.
- After upgrading, run **Refresh Metadata** once on any game whose images are missing — the fix is self-healing on the first refresh.
