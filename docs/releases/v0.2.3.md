# v0.2.3 Release Notes

`v0.2.3` expands MGA's web-first device control into a unified multi-route
library with managed local installation and emulator setup.

## Highlights

- Added Library and Play grouping, clearer cover badges, dedicated browser and
  cloud shelves, stable server-side sorting, and contextual Play controls that
  keep every available local, browser, storefront, and cloud route visible.
- Added version-aware game identity. MGA preserves source copies and concrete
  editions instead of combining games from title matching alone.
- Added automatic and manual device inventory scans with storage, recognized
  applications, emulator runtimes, RetroArch cores, firmware readiness, and
  package-manager discovery.
- Added transactional ZIP, 7z, and RAR installation, live download/install
  progress, executable discovery, installed-game validation, guarded cleanup,
  uninstall, and Play on a selected device/OS user.
- Added locally confirmed GOG Inno installation and publisher-owned uninstall,
  including recovery for installers that finish successfully and then crash.
- Added Settings > Emulators with multiple compatible emulators per platform,
  device-specific defaults, RetroArch core choices, RetroAchievements facts,
  firmware status, and allow-listed install/update actions through MGA Client.
- Added typed ScummVM and RetroArch launch paths. Server-controlled requests
  cannot supply executable paths, shell commands, package IDs, or arbitrary
  arguments to MGA Client.
- Simplified player-facing Settings language while keeping technical evidence
  available where it is useful.

## Upgrade and migration notes

- Migrations 13-24 are additive and run through MGA's backup-first migration
  flow. The updater preserves configuration, credentials, pairing identity,
  media, source cache, installation records, and SQLite data.
- Migration 13 adds version-aware title and edition identity while preserving
  existing canonical game IDs as edition IDs.
- Migrations 14-16 add endpoint inventory, command progress, managed
  installation state, and launch candidate metadata.
- Migrations 17-20 add GOG installation, cleanup, recovery, and verification
  state. Existing legacy attention rows without a cleanup marker remain
  protected from direct deletion.
- Migration 21 adds per-device/OS-user installation-root preferences.
- Migrations 22-24 add emulator defaults, RetroArch core preferences, and
  persisted package-manager inventory facts.
- These migrations are one-way. MGA creates a database backup before applying
  them and the packaged updater restores the previous application and database
  if migration validation fails.
- Trusted-LAN HTTP remains supported. MGA Server may run on another LAN device;
  this release does not require localhost or HTTPS.
