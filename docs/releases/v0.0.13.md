# v0.0.13 Release Notes

`v0.0.13` is a library-management and metadata correctness release after `v0.0.12`.

## Highlights

- Library List view bulk actions now work on every eligible selected row instead of requiring the entire selection to support the action.
- Bulk Reclassify and Bulk Hard Delete show a Continue/Cancel review dialog when some selected games will be skipped.
- Batch hard-delete preview failures no longer block deleting other successfully previewed source records; skipped records show their source label and error clearly.
- LaunchBox external links no longer build invalid public detail URLs from LaunchBox metadata-package IDs.
- Manual reclassify now carries the selected provider's full media list into the persisted source match, instead of keeping only one representative image.
- Game detail no longer duplicates source file listings in a separate Game Files section; source-owned files remain under Source Records.
- Library bulk-selection controls remain sticky below the app header while scrolling.

## Upgrade Notes

- `NO_MIGRATION_NEEDED`: v0.0.13 changes frontend workflows, metadata link construction, and manual-review media payloads only. No SQLite schema, persisted config, or sync payload changes are introduced.
- Existing installs can upgrade normally through the installer or portable update flow.
- If a LaunchBox match was previously saved with an invalid public details URL, opening the game detail page after upgrade will present the corrected LaunchBox search link. Metadata refresh or reclassify can be used to repersist improved media where needed.
