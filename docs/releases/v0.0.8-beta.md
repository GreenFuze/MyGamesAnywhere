# v0.0.8-beta Release Notes

`v0.0.8-beta` is a validation release for the new profile, installer, and update flows. It is intended for testing on a second machine before promoting the same release line to a stable `v0.0.8`.

## Highlights

- Added local profiles with first-run setup, profile picker, header profile menu, logout, and Settings -> Profiles management.
- Made configured integrations and library data profile-owned, including storefront credentials, source scans, favorites, achievements through the profile-owned library path, and profile-level frontend state.
- Added Windows installer packaging with for-me-only process installs, all-users service installs, scope-based listen defaults, tray startup, LAN/firewall setup, rotating file logs, and uninstall cleanup that keeps user data by default.
- Added runtime path separation for installed layouts so mutable data moves outside the immutable app directory.
- Added update manifest support, update status/check/download/apply APIs, About page update controls, and SHA256 verification for downloaded update assets.
- Added SemVer prerelease-aware update comparison, so `v0.0.8-beta` correctly detects `v0.0.8` as newer.
- Improved large-library loading with bounded paged Library requests and a Load More flow.
- Improved `/api/stats` performance by using aggregate queries instead of loading the whole library.

## Upgrade Notes

- Portable installs remain self-contained and writable. Back up `config.json`, `data/`, and `media/` before manually replacing portable files.
- Installed layouts keep mutable data and logs under `%LOCALAPPDATA%\MyGamesAnywhere` for per-user installs or `%ProgramData%\MyGamesAnywhere` for all-users/service installs.
- Existing single-user data is migrated into a default Admin Player profile.
- Anyone who can access the local web UI can choose or switch profiles in this beta. Profiles are convenience identities, not password-protected accounts.
- Portable auto-update v1 downloads and verifies the ZIP but does not replace the running folder automatically.

## Validation Focus

- Fresh install first-run profile creation.
- Existing portable data migration into the default Admin Player profile.
- For-me-only installer mode, which starts MGA after that Windows user signs in.
- All-users installer mode, which runs MGA as a Windows service.
- Tray start/stop/open behavior for for-me and service modes.
- For-me `127.0.0.1` binding, all-users `0.0.0.0` binding, and firewall rule behavior.
- About page update check/download/apply behavior.
- Portable ZIP download verification and manual replacement guidance.
