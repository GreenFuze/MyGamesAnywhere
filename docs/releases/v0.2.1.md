# v0.2.1 Release Notes

`v0.2.1` is a trusted-LAN transport compatibility patch for the web-first MGA
Server and standalone MGA Client introduced in `v0.2.0`.

## Changes

- Profile password/PIN login now works when the MGA web interface is opened
  over HTTP from another computer on the LAN.
- Profile credential changes and authenticated device management use the same
  trusted-LAN HTTP policy.
- MGA Client pairing, signed `mga://start` acknowledgement, and live device
  connections now support HTTP/WS server URLs outside loopback.
- HTTPS/WSS remains fully supported and is recommended whenever the network is
  not fully trusted.
- Added regression coverage for non-loopback HTTP login, authenticated device
  sessions, pairing/launch handlers, a real WS handshake, and MGA Client LAN URL
  validation.

## Security

HTTP/WS avoids browser certificate warnings and local certificate deployment,
but it does not transport-encrypt profile credentials, session cookies,
pairing data, or device traffic. Use it only on a trusted LAN. Do not expose an
HTTP MGA Server directly to the internet; use HTTPS/WSS with a trusted
certificate for untrusted or routed networks.

## Upgrade Notes

- `NO_MIGRATION_NEEDED`: this patch changes transport acceptance and
  documentation only. Existing SQLite data, client schema-1 JSON, protected
  endpoint keys, and server configuration remain compatible and unchanged.
