# v0.2.2 Release Notes

`v0.2.2` simplifies optional profile credentials for MGA's trusted-LAN usage.

## Changes

- Passwords now accept any four or more Unicode characters, with no uppercase,
  lowercase, number, symbol, or other composition requirement.
- PINs now accept four or more digits, with no maximum length.
- Profile setup, credential changes, and forced bootstrap-password replacement
  use the same policy in both the web interface and server validation.
- Added boundary coverage for short and valid passwords/PINs, unrestricted
  password characters, Unicode character counting, long PINs, and digits-only
  PIN enforcement.

## Upgrade Notes

- Existing credentials and their Argon2 hashes remain valid and are not reset.
- `NO_MIGRATION_NEEDED`: only validation of newly initialized or changed
  credentials is relaxed. Existing sessions, SQLite data, MGA Client schema-1
  JSON, endpoint keys, and server configuration remain unchanged.
